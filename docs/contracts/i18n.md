# i18n contract ŌĆö FR/EN (F13)

`@canvas/i18n` is the single i18n foundation for LivePlace. Every visible MVP
string flows through it. Scope is **FR + EN only**; user-generated content
(canvas titles) is **not** translated.

## Packages / entries

- `@canvas/i18n` ŌĆö framework-agnostic core (types, detection, persistence,
  catalogs, observable store). No React dependency; usable from the OBS view and
  server-side.
- `@canvas/i18n/react` ŌĆö React bindings (`I18nProvider`, `useI18n`, `useLocale`,
  `useTranslate`, `LanguageSwitcher`).

## Adding a UI string

1. Add the key + English text to `packages/i18n/src/messages/en.ts` (the source
   of truth ŌĆö its keys define `MessageKey`).
2. Add the same key to `fr.ts`. The compiler **rejects** any catalog that is
   missing or adds a key, so FR/EN parity is enforced at build time. A runtime
   test (`test/i18n.test.ts`) also asserts parity + non-empty values.
3. Use it: `const t = useTranslate(); t("area.thing", { count: 3 })`.

Keys are flat and dot-namespaced (`area.thing`). Placeholders are `{name}` and
are filled by `t(key, params)`. No ICU/plural rules in the MVP ŌĆö keep strings
simple. **Never hardcode user-facing text.**

## Locale selection

`detectInitialLocale({ stored, twitchLocale, navigatorLanguages, fallback })`
resolves the locale by precedence, first match wins:

1. `stored` ŌĆö the persisted preference (`profiles.locale` if authenticated,
   else the locale cookie). An explicit choice must win (CA2).
2. `twitchLocale` ŌĆö from the authenticated Twitch profile (wired by FEN-11).
3. `navigatorLanguages` ŌĆö `navigator.languages`, in order.
4. `fallback` ŌĆö `DEFAULT_LOCALE` (`en`).

All candidates are normalised, so `fr-FR`, `en_US`, etc. resolve; unsupported
tags are skipped, not forced to the fallback.

## Switching without reload (CA1)

The store is observable. `setLocale` / `toggle` mutate in place and notify
subscribers synchronously; React binds via `useSyncExternalStore`, so a switch
re-renders the tree ŌĆö **no page reload**. `<LanguageSwitcher />` is the ready
component.

## Persistence (CA2)

Pass an `onChange` to `createI18n` to persist. `apps/web/src/i18n.ts` writes the
`liveplace_locale` cookie (1 year, `SameSite=Lax; Secure`) and reflects
`<html lang>`.

**Authenticated users:** the auth/profile layer (FEN-11 / FEN-22) must also
persist to `profiles.locale` and feed it back as `stored` on the next load, so
the preference follows the account across devices. Implement the
`LocalePersistence` interface (`load()/save()`) with a Convex-backed version;
keep mirroring to the cookie so the first paint (before the session resolves) is
correct.

## Cookie

- Name: `liveplace_locale` (exported as `LOCALE_COOKIE`).
- Not HttpOnly ŌĆö the client and SSR/OBS view read it to pick the initial locale
  before any round-trip. It holds only a locale code (`en`/`fr`), no PII.
